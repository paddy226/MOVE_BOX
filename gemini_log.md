# Move Box 專案開發紀錄 (2026-03-19) 第二十三次更新

## 1. 今日更新摘要
- **挑戰模式：進度記憶與自動導航 (Progress Persistence & Auto-Navigation)**：

    - **圖示放大**：提升了進度圖示的尺寸 (80x80) 並調整透明度，兼顧視覺張力與資訊可讀性。

# Move Box 專案開發紀錄 (2026-03-19) 第二十二次更新

## 1. 今日更新摘要
- **挑戰模式：離線優先同步機制 (Offline-First Sync Engine)**：
    - **本地快取系統**：引入了本地清單管理 (`challenges_manifest.json`)，將雲端關卡持久化儲存於 `user://challenges/` 目錄。
    - **智慧增量同步**：進入挑戰選單時，自動比對 GitHub API 的 SHA 雜湊值。僅下載新增或內容有變動的關卡，大幅節省流量並縮短載入時間。
    - **完全離線支援**：在無網路環境下，系統會自動切換至「離線模式」，直接讀取已快取的本地關卡，確保遊玩不中斷。
- **UI 與 效能優化**：
    - **下一關載入優化**：通關後的「下一關」按鈕改為從本地讀取，消除了原本需要即時連網下載的卡頓感。
    - **同步視覺修補**：修正了同步狀態文字 (`Syncing...`) 在完成後殘留並與按鈕重疊的問題。

## 2. 當前版本資訊

# Move Box 專案開發紀錄 (2026-03-17) 第二十一次更新

## 1. 今日更新摘要
- **編輯器：UI 佈局大重構 (UI Layout Overhaul)**：
    - **分區工具列**：將工具列拆分為兩部分：右上角專屬「匯入/分享」區，右下角為「管理/測試」區。徹底解決按鈕過多導致與返回按鈕重疊的問題。
    - **視覺簡化**：隱藏了右上角的鍵盤操作說明文字 (HelpLabel)，並將頂部提示簡化為單行顯示 (`Tool: [名稱]`)，使編輯介面更清爽。
- **Android：網路連線修正與升級 (Network & Engine Fixes)**：
    - **權限補全**：修正了 Android 版本無法載入挑戰關卡（Error 0）的問題，正式開啟 `INTERNET` 與 `ACCESS_NETWORK_STATE` 權限。
    - **引擎升級測試**：成功在 **Godot v4.6.1-stable** 環境下完成 Android APK 匯出與部署測試，確認相容性良好。
- **Android 包名規範化**：
    - 確立 Android 包名為 `com.example.movebox`，優化了自動化安裝與啟動流程。
