# Move Box 專案開發紀錄 (2026-03-20) 第二十五次更新

## 1. 今日更新摘要
- **檔案管理：捲動式清單系統 (V2 Scrolling System)**：
    - **Custom 模式升級**：廢棄分頁模式，改為支援搜尋與排序的長列表。實作了「滑鼠拖曳捲動」與「防誤觸位移偵測 (Threshold)」，在 PC 與手機上均有優異體驗。
    - **智慧搜尋與排序**：新增關鍵字搜尋框，並支援按「修改日期 (Date)」或「檔案名稱 (Name)」排序，系統預設顯示最新編輯的檔案。
- **編輯器：UI 安全與模態彈窗 (Modal UI & Safety)**：
    - **模態遮罩層 (Modal Dimmer)**：實作全螢幕半透明遮罩，當任何彈窗出現時，自動攔截背景 HUD 按鈕點擊，徹底解決誤觸問題。
    - **清除確認機制**：為編輯器「清除畫布」按鈕加入二度確認視窗，防止心血意外流失。
    - **載入介面統一**：將編輯器的「載入檔案」介面同步升級為 V2 捲動系統，操作感與 Custom 模式完全一致。
- **UI 修復與層級調整**：
    - 修正了 `editor.tscn` 中的節點層級順序，確保遮罩層正確位於 HUD 之上、彈窗之下。
    - 修正了 `DeletePopup` 的節點路徑引用錯誤。

# Move Box 專案開發紀錄 (2026-03-19) 第二十四次更新

## 1. 今日更新摘要
- **UI 全面分頁化 (Universal Pagination)**：
    - **Custom 模式優化**：為解決自定義關卡過多時捲動不便的問題，將 `Custom` 模式改為與挑戰模式一致的分頁介面（每頁 10 關）。
    - **編輯器載入面板優化**：編輯器內的「載入檔案」功能同步改成分頁模式，並統一了刪除按鈕與佈局，大幅提升在行動裝置上的操作流暢度。
    - **刪除確認強化**：將刪除檔案時的彈出視窗文字與按鈕字體加大，確保在高解析度螢幕下仍能輕鬆操作。
- **挑戰模式：漸進式同步優化 (Progressive Syncing)**：
    - **優先隊列機制**：進入挑戰模式時，改為優先同步「玩家當前分頁」的 10 個關卡。
    - **背景靜默同步**：介面顯示後，剩餘的關卡會在背景默默下載，不再讓玩家在進入選單時長時間等待，實現「秒開」體驗。
- **Bug 修正**：
    - 修復了編輯器載入視窗中 `CANCEL` 按鈕失效的問題。
    - 統一了所有按鈕的觸控事件穿透設定 (`mouse_filter = PASS`)。

- **更新重點**：全介面分頁系統、背景靜默同步、編輯器 UI 優化、刪除確認強化。

# Move Box 專案開發紀錄 (2026-03-19) 第二十二次更新

## 1. 今日更新摘要
- **挑戰模式：離線優先同步機制 (Offline-First Sync Engine)**：
    - **本地快取系統**：引入了本地清單管理 (`challenges_manifest.json`)，將雲端關卡持久化儲存於 `user://challenges/` 目錄。
    - **智慧增量同步**：進入挑戰選單時，自動比對 GitHub API 的 SHA 雜湊值。僅下載新增或內容有變動的關卡，大幅節省流量並縮短載入時間。
    - **完全離線支援**：在無網路環境下，系統會自動切換至「離線模式」，直接讀取已快取的本地關卡，確保遊玩不中斷。
- **UI 與 效能優化**：
    - **下一關載入優化**：通關後的「下一關」按鈕改為從本地讀取，消除了原本需要即時連網下載的卡頓感。
    - **同步視覺修補**：修正了同步狀態文字 (`Syncing...`) 在完成後殘留並與按鈕重疊的問題。
