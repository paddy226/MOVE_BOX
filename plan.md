# 3D 翻箱子 (Rolling Box Puzzle) 開發計畫

## 1. 核心機制 (Core Mechanics)
- **翻滾控制 (Rolling Movement)**：
    - 箱子以邊緣為軸心進行 90 度翻轉，而非平滑移動。
    - 限制箱子只能在網格內移動。
- **操作方式 (Controls)**：
    - **滑動操作 (Swipe Control)**：支援手機遊玩。玩家手指滑動一下，箱子即往滑動方向翻轉一次。
    - **鍵盤操作 (Keyboard)**：作為備援或桌面端測試（WASD / 方向鍵）。
- **面數管理 (Face Value System)**：
    - 定義箱子六個面的數字 (1-6)。
    - 即時追蹤目前哪一個面朝下（與地面接觸）。
- **顏色系統 (Color System)**：
    - 箱子具有顏色屬性（例如：紅色、綠色、藍色）。
    - 玩家翻滾至目標格子時，必須滿足：**[朝下的面數字 == 目標數字] 且 [箱子顏色 == 目標顏色]**。
- **換色點 (Color Changer)**：
    - 地圖中的特殊格子。
    - 箱子踩上後會改變顏色。
    - **次數限制**：
        - 無限次數：標記為 `∞`。
        - 有限次數：標記為剩餘次數，每踩過一次數字減 1，歸零後失效。
- **目標檢測與完成狀態 (Goal Matching & Completion)**：
    - 當箱子完全符合數字與顏色要求時，該點視為完成。
    - **完成回饋**：完成時目標點必須有明確的視覺變化（例如：變為固定顏色如金色、發光、或出現打勾符號）。
- **過關條件 (Win Condition)**：
    - 場景內所有目標點皆完成。

## 2. 階段性開發計畫 (Milestones)

### 階段一：基礎原型 (Protoryping)
- [ ] 建立基礎地板格子與不規則地形系統。
- [ ] 實作箱子翻轉動畫與邏輯（以邊緣為軸心旋轉）。
- [ ] **操作系統**：實作滑動偵測（Swipe Detection）邏輯。
- [ ] 建立箱子六面數字與顏色狀態。

### 階段二：邏輯與判定 (Logic & Validation)
- [ ] 判定目前朝下的面數值。
- [ ] 實作目標格子（Goal Tile）與數字/顏色對比邏輯。
- [ ] 實作換色點（Color Changer）邏輯與次數管理。
- [ ] 視覺回饋：實作目標點完成時的固定顏色/特效顯示。

### 階段三：關卡與系統 (Level & Systems)
- [ ] 實作簡易關卡載入。
- [ ] 實作「上一步/復原 (Undo)」功能（益智遊戲必備）。
- [ ] 過關 UI 與重啟機制。

## 3. 技術規格 (Technical Specs)
- **引擎版本**: Godot 4.6.1 (GL Compatibility)
- **物理引擎**: Jolt Physics
- **渲染器**: Direct3D 12 (Windows) / OpenGL ES 3.0 (Mobile)

## 4. 開發規範 (Development Standards) [NEW]
- **模組化拆分**：如果程式碼行數過多，應將功能拆分成多個子程式（例如：將滑動偵測、翻轉邏輯、顏色管理拆分），保持結構清晰。
- **完整性優先**：不要主動精簡或壓縮程式碼，應優先確保程式的可讀性與邏輯完整性。
- **異動確認**：已經實作完成的功能，如果後續需要修改，**必須先與使用者確認**，嚴禁擅自更動已完成的部分。
